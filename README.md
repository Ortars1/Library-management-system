# 📚 Управление библиотекой (Library Management System)

Веб-приложение для автоматизации учёта книжного фонда, выдачи и возврата книг. Разработано в рамках курсовой работы по дисциплине **«Разработка и проектирование клиент-серверных приложений»**. Приложение построено по монолитной клиент-серверной архитектуре с контейнеризацией и ориентировано на простоту развёртывания, прозрачность кода и соответствие современным стандартам RESTful-разработки.

---

##  Функциональные возможности

| Модуль | Описание |
|--------|----------|
| 📖 Каталог | Просмотр всех книг, поиск по названию и автору (регистронезависимый) |
| 🔐 Аутентификация | Регистрация, вход, JWT-токены, защита маршрутов |
| 📥 Выдача книг | Оформление ссуды читателем, атомарное уменьшение остатков |
| 📤 Возврат книг | Приём возвратов библиотекарем/админом, инкремент остатков |
| 👤 Личный кабинет | Активные выдачи и архив возвращённых книг (только для читателей) |
| 📚 Управление фондом | Добавление и удаление книг, просмотр активных выдач |
| ⚙️ Админ-панель | Просмотр пользователей, смена ролей, удаление учётных записей |

**Ролевая модель:**
- `guest` — просмотр каталога и поиск
- `reader` — взятие книг, просмотр личных выдач
- `librarian` — управление фондом, приём возвратов
- `admin` — полное управление, назначение ролей, удаление пользователей

## 🛠 Стек технологий

| Слой | Технология |
|------|------------|
| **Backend** | Python 3.11+, FastAPI, SQLAlchemy 2.0, Pydantic, Passlib+bcrypt, python-jose |
| **Frontend** | Jinja2 (SSR), Vanilla JS + Fetch API, CSS3 |
| **Database** | PostgreSQL 15 |
| **DevOps** | Docker, Docker Compose |
| **API** | RESTful, OpenAPI/Swagger (`/docs`) |

## 🚀 Запуск проекта

Требования: установленные `Docker` и `Docker Compose`.

```bash
# Клонирование репозитория
git clone <ссылка-на-репозиторий>
cd library-app

# Запуск контейнеров
docker-compose up --build

# Приложение доступно по адресу:
# http://localhost:8000
# Swagger-документация API: http://localhost:8000/docs
```

При первом запуске автоматически создаётся схема БД и учётная запись администратора.

## 📜 Примечание

Данная версия представляет собой **учебный прототип**, оптимизированный под требования курсовой работы. В целях минимизации скоупа исключены: пагинация каталога, адаптивная вёрстка, сложные миграции (таблицы создаются автоматически при старте), email-уведомления и внешние каталоги. Система полностью покрывает заявленные сценарии использования, обеспечивает транзакционную целостность операций и готова к демонстрации в рамках академической отчётности.
